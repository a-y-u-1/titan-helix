Every enterprise sits on a graph it can't see.

TITAN HELIX rebuilds that graph from raw telemetry every few seconds, runs a multi-agent debate over what's actually wrong, forecasts where the failure is heading, and recommends action — with the SPL trail to prove it.

It uses Splunk as long-term memory and an agent mesh as the mind. This package is the runnable foundation: live + historical telemetry, the Splunk backbone, a reasoning layer, and the command-center visuals.

Quick start · 5 steps

1

Set up everything

./setup.sh

Installs deps + Docker, starts Splunk, creates indexes, configures HEC.

2

Load 7-day history

python3 historical_generator.py --days 7 --output data/history.jsonl python3 load_to_splunk.py data/history.jsonl --hec-url … --hec-token …

Baselines + 3 past incidents the Memory agent recalls.

3

Stream the cascade

python3 synth_generator.py --hec-url … --hec-token … \ --scenario scenarios/checkout_collapse.yaml --speed 10

Live firehose + a 6-phase cascading failure.

4

Run the agents

python3 agents.py --investigate checkout-api

Reasoning chain over real Splunk data. --mock-data to test with no infra.

5

Open the UI

uvicorn backend.app:app --host 0.0.0.0 --port 8080

Command center at localhost:8080, served from live Splunk data.
